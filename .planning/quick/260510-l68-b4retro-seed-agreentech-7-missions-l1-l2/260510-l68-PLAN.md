-- No assumption that total_score ∈ [0..100]. Safe with max_score=25.
with best_per_template as (
  select s.deliverable_template_id, max(e.total_score) as max_score
  from public.submissions s
  join public.evaluations e on e.submission_id = s.id
  where s.player_id = p_player_id and s.status = 'validated'
  group by s.deliverable_template_id
)
select coalesce(sum(max_score), 0) into v_total from best_per_template;

**Rationale** :
- Le bound effectif par critère reste la loop `value > criterion.max` (ligne ~456) — c'est elle qui catche correctement >5 par critère.

Choisir `.max(25)` (>= max criterion) = compatible avec toute rubric AgriTech même hypothétique 1×25 (pour future-proof) ; choisir `.max(5)` = strict actuel mais casserait un futur ajustement (ex: `quality:10`). **Recommandation : `.max(25)`** (matches max_score global, future-proof, defense-in-depth maintenue).
