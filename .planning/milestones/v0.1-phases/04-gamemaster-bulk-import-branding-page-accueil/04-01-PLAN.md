  <action>
Create `lib/admin.ts` modeled on `lib/mentor.ts` (server-side, dual-mode safe). Export:

```typescript
export type CohortRow = {
  player: Player;
  levelLabel: string;
  status: "en_avance" | "a_l_heure" | "retard"; // derived
  nextDeliverableTitle: string | null; // first non-validated deliverable_template by ord/scheduled
};

export type GlobalCounters = {
  totalSubmissions: number;
  pendingReview: number; // status in ('submitted_v1','submitted_v2') AND no evaluation row yet
  validated: number;     // status in ('validated')
  totalDeliverableSlots: number; // players * deliverable_templates of event
};

export async function getCohortOverview(): Promise<CohortRow[]>;
export async function getGlobalCounters(): Promise<GlobalCounters>;
```

Implementation:
1. Resolve current event: `events` order by `starts_at desc limit 1` (mirror mentor.ts).
2. `getCohortOverview()`:
   - Load cohorts of event, players of cohorts (ordered by name).
   - Load all `missions` of event with `id, ord, level_id, scheduled_at`, then their `deliverable_templates` (`id, mission_id, title, ord`).
   - Build ordered list of deliverable_templates (sort by mission ord ASC then template ord ASC).
   - Load all submissions `id, player_id, deliverable_template_id, status` for those players.
   - For each player, "next deliverable" = first template whose latest submission status is NOT in ('validated', 'submitted_v1', 'submitted_v2') OR has no submission. If all done, `null`.
   - Status derivation (pilot-grade, document choice in code comment):
     * `en_avance`: validated count > expected_for_now (expected = floor(missions whose scheduled_at <= now / total) * total templates) — practically: validated_count >= 2 ahead of expected.
     * `retard`: validated_count < expected - 2 OR has no submission for any past-scheduled mission.
     * `a_l_heure`: otherwise.
     For Day-1 simplicity, use: `retard` if `submittedCount === 0 && elapsedMissions > 0`; `en_avance` if `validatedCount >= elapsedMissions + 1`; else `a_l_heure`. Add a clear code comment that this is pilot heuristic.
   - `levelLabel` via `levelLabel(player.currentLevel)`.
3. `getGlobalCounters()`:
   - `totalSubmissions` = count submissions for players of event.
   - `pendingReview` = count submissions in ('submitted_v1','submitted_v2') with no row in evaluations (LEFT JOIN, or in-memory diff via two queries).
   - `validated` = count submissions where status='validated'.
   - `totalDeliverableSlots` = playersCount * templatesCount.
4. Dual-mode: `if (!supabase) return [] / zeros;` like mentor.ts. Log errors via `console.error("[admin] ...")` and return safe defaults — no throws.

Map snake_case rows to camelCase Player (mirror `mapPlayer` in lib/mentor.ts).
