**Étapes** :

1. **Login Mentor** sur `/login` → redirige vers `/mentor`.
   - ✅ Liste des Players visible avec colonnes Equipe / Idee / Niveau / Score Projet / Soumis/Total / En attente.
   - ✅ Filtre "Afficher uniquement Players en attente" toggle l'URL `?pending=1` et restreint la liste.

2. **Cliquer "Evaluer"** sur un Player avec pending → `/mentor/submission/[id]`.
   - ✅ Header montre équipe, idée, score Projet courant.
   - ✅ Bloc Soumission montre version=1, lien proof_url cliquable (target=_blank) ou texte preuve.
   - ✅ Form rubric : 1 input par critère, max correspondant. Total live calculé.
   - ✅ Textarea feedback. 3 boutons : Valider V1 / Demander V2 / Rejeter.

3. **Soumettre verdict request_v2** avec scores (ex: 30/40, 25/30, 15/20, 5/10), feedback non vide.
   - ✅ Page recharge avec banner "Vous avez deja evalue cette soumission." si rouverte.
   - ✅ Vérifier en SQL : `select * from evaluations where submission_id='...'` → row présente, scores JSONB, total_score=somme.
   - ✅ `select status from submissions where id='...'` → 'feedback_received'.
   - ✅ `select score_project from players where id='...'` → INCHANGE (V1 non validée).

4. **Logout, login Player** propriétaire de la submission.
   - ✅ `/journey` → statut deliverable = "Feedback recu".
   - ✅ `/journey/deliverable/[id]` → carte feedback Mentor (verdict V2 demandee, total, scores breakdown, message), puis form "Soumettre la V2".

5. **Player soumet V2** (URL https://example.com/v2 ou texte ≥10 chars).
   - ✅ Message "Soumission V2 enregistree."
   - ✅ `/journey` → statut "Soumis V2".
   - ✅ `select * from submissions where player_id=... and deliverable_template_id=... order by version` → 2 rows (V1 status=feedback_received, V2 status=submitted_v2).

6. **Logout, login Mentor**, retour `/mentor`.
   - ✅ Player a "En attente" ≥1 (la V2 nouvelle).
   - ✅ Cliquer Evaluer → /mentor/submission/[v2-id].
   - ✅ Form propose UNIQUEMENT validate_v2 et reject (pas request_v2).

7. **Mentor verdict validate_v2** (scores complets).
   - ✅ Action OK.
   - ✅ `select status from submissions where id='[v2-id]'` → 'validated'.
   - ✅ `select score_project from players where id='...'` → ÉGAL au total_score V2 (trigger Postgres recalc).

8. **Login Player** → `/journey` montre statut "Valide" et score Projet à jour.

9. **Anti-régression** : 2e Mentor (si dispo) ouvre `/mentor/submission/[v2-id]` → doit voir le readonly "Vous avez deja evalue" SI c'était le même mentor ; sinon voit form vide. **Tester** : un seul Mentor ne peut PAS évaluer 2× la même Submission (UI + DB unique constraint).

10. **Audit final** : `npm run lint`, `npm run typecheck`, `npm run build` → 0 erreur, 0 warning suspect.
