```markdown
# Deploiement prod - Entrepreneur Game

## Pre-requis
- Compte Vercel (free tier suffit)
- Projet Supabase prod cree avec schema applique (database/schema.sql -> triggers.sql -> rls.sql)
- Domain optionnel custom EIC

## Etapes initiales (premier deploiement)
1. `vercel link` (CLI) ou import via dashboard depuis GitHub repo
2. Configurer env vars Vercel :
   - `NEXT_PUBLIC_SUPABASE_URL` (Supabase project URL)
   - `NEXT_PUBLIC_SUPABASE_ANON_KEY` (anon key)
   - `SUPABASE_SERVICE_ROLE_KEY` (service role - secret) - pour Production env uniquement
   - `NEXT_PUBLIC_SITE_URL` (URL Vercel ou custom)
3. Configurer Supabase Auth :
   - Site URL : URL Vercel (ou custom)
   - Redirect URLs : ajouter URL Vercel + custom
   - Email templates magic link : verifier locale FR
4. Push sur main -> deploiement auto.

## Procedure deploy continu
- Push sur main -> Vercel build + deploy auto
- Preview deploys sur PR (tester avant merge)

## Rollback
- Vercel dashboard -> Deployments -> Promote previous deployment to production
- Si schema DB casse : restaurer un snapshot Supabase (Dashboard -> Database -> Backups)

## Variables critiques
| Var | Where | Public | Notes |
|---|---|---|---|
| NEXT_PUBLIC_SUPABASE_URL | Build + Runtime | Yes | URL Supabase |
| NEXT_PUBLIC_SUPABASE_ANON_KEY | Build + Runtime | Yes | RLS-protected |
| SUPABASE_SERVICE_ROLE_KEY | Runtime server | NO - secret | Bypass RLS, server only |
| NEXT_PUBLIC_SITE_URL | Build + Runtime | Yes | Pour redirects magic link |

## Domain custom
1. Vercel -> Settings -> Domains -> Add `entrepreneur.eic.uemf.ac.ma` (exemple)
2. Configurer CNAME chez DNS UEMF
3. Mettre a jour Supabase Auth Site URL et NEXT_PUBLIC_SITE_URL.

## Smoke test post-deploy
Voir `.planning/phases/05-.../SMOKE-TEST-E2E.md`.
```
