<task type="auto">
  <name>Task 1: Create lib/admin-player-detail.ts data layer</name>
  <files>lib/admin-player-detail.ts</files>
  <action>
Create server-side module mirroring `lib/mentor.ts` patterns.

Exports:
```typescript
export type PlayerDetailMember = {
  member: PlayerMember;
  email: string | null;
  fullName: string | null;
};
export type PlayerDetailSubmission = {
  submission: Submission;
  templateTitle: string;
  evaluations: Evaluation[]; // 0..N
};
export type PlayerDetail = {
  player: Player;
  levelLabel: string;
  members: PlayerDetailMember[];
  submissions: PlayerDetailSubmission[]; // ordered by submitted_at desc
};
export async function getPlayerDetail(playerId: string): Promise<PlayerDetail | null>;
```

Implementation:
1. `createClient()` ; if null -> return null.
2. Fetch player row by id ; if absent -> null.
3. Fetch `player_members` where `player_id = X`. Then fetch `profiles` where `user_id in (...)` to resolve email + full_name. Map.
4. Fetch `submissions` where `player_id = X` ordered by `submitted_at desc`. Then `deliverable_templates` where `id in (template_ids)` to resolve titles into a Map. Then `evaluations` where `submission_id in (sub_ids)`.
5. Build `submissions: PlayerDetailSubmission[]` with title + evaluations bucketed.
6. Map snake_case to camelCase consistently (mirror `lib/mentor.ts:mapPlayer`).
7. `levelLabel(player.currentLevel)` from `lib/journey.ts`.
8. Wrap each query with try-style error logging `console.error("[admin-player-detail] ...")` and return null/[] safely on errors. Never throw.

UUID validation: if `playerId` is not a valid UUID format, return null early (avoid DB roundtrip on bad input).
