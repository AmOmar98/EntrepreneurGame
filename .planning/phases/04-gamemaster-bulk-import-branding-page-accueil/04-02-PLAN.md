  <files>app/actions.ts</files>
  <action>
Append to `app/actions.ts`:

```typescript
import { parseCsv, dedupeCsvRows, slugifyTeam, type ImportReport } from "@/lib/admin-import";

export type ImportWorkflowState = WorkflowState & { report?: ImportReport };

const importSchema = z.object({
  csvText: z.string().min(10).max(200_000),
  cohortSlug: z.string().min(2).max(64).default("hack-days-mai-2026"),
});

export async function importPlayersCsv(
  _prev: ImportWorkflowState,
  formData: FormData,
): Promise<ImportWorkflowState>;
```

Logic:
1. Authorize: `getCurrentUser()` + `getCurrentRole()`. If role !== "game_master" -> `{ ok: false, message: "Forbidden" }`.
2. Validate `csvText` via Zod.
3. `hasSupabaseEnv()` guard — return `{ ok: false, message: "Auth backend not configured." }` else.
4. `parseCsv` -> `dedupeCsvRows`. Initialize `report: ImportReport`.
5. Resolve current event (latest by `starts_at`) and resolve/upsert cohort by slug for this event:
   - Look up `cohorts where event_id = X and slug = ?`. If absent, insert with `name = "Hack-Days Mai 2026"` (default). Idempotent.
6. For each row:
   - Compute base slug `slugifyTeam(teamName)`. Find player `where slug = base`. If exists with same `name` & `cohort_id` -> `alreadyExisted++`, reuse id. Else if slug taken by another team in another cohort -> append `-{cohortSlug}` to slug. Insert player with `cohort_id, slug, name=teamName, idea=projectPitch`. `created++`.
   - For `[leaderEmail, ...memberEmails]`:
     a) Look up `auth.users` by email via Admin API IF service role available, else look up `profiles where email = ?`. If found -> ensure `player_members(player_id, user_id)` row exists (unique constraint = idempotent). `membersAdded++` only when actually inserted.
     b) If user not found AND service-role configured: `await supabase.auth.admin.inviteUserByEmail(email, { data: { team_name: teamName } })`. On success -> `invitesSent++`, then re-resolve user_id and insert player_member. On error -> `errors.push({ email, reason: error.message })`.
     c) If service-role NOT configured AND user missing: `invitesSkipped++`, push error `"service role missing — invite skipped"`.
   - Mark leader's `player_member.team_role = 'owner'`; others `'contributor'`.
   - On the leader row, also upsert `profiles(user_id, app_role='player', email=leaderEmail)` so role-gating works at first login.
7. Service-role detection: read `process.env.SUPABASE_SERVICE_ROLE_KEY`. If present, build a separate server-only admin client via `createClient(SUPABASE_URL, SERVICE_ROLE_KEY, { auth: { persistSession: false } })` from `@supabase/supabase-js`. Do NOT leak this client outside the action. Add `SUPABASE_SERVICE_ROLE_KEY` to `.env.example` (one line).
8. Each Supabase error -> push to `report.errors` with `{ email?, reason }`. Never throw.
9. `revalidatePath("/admin")`, `revalidatePath("/admin/players/import")`.
10. Return `{ ok: report.errors.length === 0, message: ..., report }`.

Idempotency proof points:
- player insert keyed by `(cohort_id, slug)` lookup-then-insert.
- player_members `unique(player_id, user_id)` -> on conflict do nothing semantics achieved via prior lookup.
- Re-running same CSV -> `created=0, alreadyExisted=N, membersAdded=0, invitesSent=0` (users already linked).

Do NOT modify the existing `signIn`, `signOut`, `saveOnboarding`, `submitDeliverable`, evaluate actions.
