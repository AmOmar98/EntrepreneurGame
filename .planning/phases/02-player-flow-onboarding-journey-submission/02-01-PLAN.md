  <name>Task 1: saveOnboarding server action + middleware redirect</name>
  <files>app/actions.ts, utils/supabase/middleware.ts</files>
  <action>
1. Dans `app/actions.ts`, ajouter le schema Zod `onboardingSchema` :
   - teamName: z.string().min(2).max(80)
   - idea: z.string().min(10).max(500)
   - q1..q5: z.coerce.number().int().min(1).max(5)
   - membersConfirmed: z.string().optional() (CSV des user_ids confirmes ; FormData.getAll alternative — cf design)
2. Exporter `saveOnboarding(_prev: WorkflowState, formData: FormData): Promise<WorkflowState>` :
   - Verifier `hasSupabaseEnv()` ; sinon return { ok:false, message:"Auth backend not configured." }
   - `createClient()` ; `supabase.auth.getUser()` ; si pas user -> { ok:false, message:"Not authenticated." }
   - safeParse formData ; sinon return { ok:false, message: parsed.error.issues[0].message }
   - Resoudre player du user via `player_members` (select player_id where user_id = user.id) ; si introuvable -> { ok:false, message:"Aucun Player rattache a votre compte." }
   - Si player.onboarded_at IS NOT NULL deja -> return { ok:true, message:"Onboarding deja complete." } (idempotent)
   - update `players` set name=teamName, idea=idea, onboarded_at=now(), score_engagement = score_engagement + 10 where id = player.id
   - revalidatePath("/journey"), revalidatePath("/onboarding")
   - return { ok:true, message:"Onboarding enregistre." }
   - Sur toute erreur Supabase, retourner { ok:false, message: error.message } (DATA-04 : pas de swallow)
3. Dans `utils/supabase/middleware.ts`, apres avoir resolu `user` :
   - Si user existe ET pathname n'est pas dans `/login`,`/onboarding`,`/api`,`/_next`,`/auth/callback` :
     - select profiles.app_role pour user.id
     - si app_role === 'player' :
       - select onboarded_at depuis players via join player_members where user_id = user.id (single ; maybeSingle)
       - si data && data.onboarded_at IS NULL -> redirect to /onboarding
   - Si user && pathname === '/onboarding' ET app_role === 'player' ET onboarded_at IS NOT NULL -> redirect to /journey
   - Mentor/game_master : ne jamais rediriger vers /onboarding
   - Garder la redirection existante /login pour user absent.

NE PAS persister les Likert/membres dans le schema (decision : pas de table dediee Phase 2). Les valeurs sont validees pour fermer l'UX mais non stockees ; le SUMMARY doit le mentionner.
