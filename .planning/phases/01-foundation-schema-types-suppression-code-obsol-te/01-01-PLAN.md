```sql
-- events: a Hack-Days instance (multi-event ready S3)
events(id uuid pk, slug text unique, name text, starts_at timestamptz, ends_at timestamptz,
       results_published_at timestamptz null, created_at, updated_at)

-- levels: 0..7 referenced by enum, label/description (S3)
levels(id level_id pk, ord smallint, label text, description text)

-- missions: unit of work for a given event (e.g. "Atelier Problem")
missions(id uuid pk, event_id fk, level_id level_id, ord smallint, kind mission_kind,
         title text, scheduled_at timestamptz null, created_at, updated_at,
         unique(event_id, level_id, ord))

-- deliverable_templates: a deliverable expected from a mission (rubric definition)
deliverable_templates(id uuid pk, mission_id fk, slug text, title text,
                      description text, rubric jsonb, max_score int, ord smallint,
                      created_at, updated_at, unique(mission_id, slug))

-- cohorts: a cohort scoped to an event
cohorts(id uuid pk, event_id fk, slug text, name text, created_at, updated_at,
        unique(event_id, slug))

-- players: 1 row per team (renamed from startups)
players(id uuid pk, cohort_id fk, slug text unique, name text, idea text null,
        current_level level_id default 'L0_diagnostic',
        status player_status default 'active',
        score_project numeric(6,2) default 0,
        score_engagement numeric(6,2) default 0,
        onboarded_at timestamptz null,
        created_at, updated_at)

-- player_members: a user belongs to a player (team)
player_members(id uuid pk, player_id fk, user_id uuid fk auth.users,
               role app_role default 'player', team_role team_role default 'contributor',
               joined_at timestamptz default now(),
               unique(player_id, user_id))

-- profiles: app_role per user (mentor/game_master live here without player_members row)
profiles(user_id uuid pk fk auth.users, app_role app_role default 'player',
         full_name text null, email text null, created_at, updated_at)

-- submissions: V1/V2 per (player, deliverable_template)
submissions(id uuid pk, player_id fk, deliverable_template_id fk,
            version smallint check (version in (1,2)),
            kind submission_kind, proof_url text null, proof_text text null,
            status submission_status default 'submitted_v1',
            submitted_by uuid fk auth.users, submitted_at timestamptz default now(),
            unique(player_id, deliverable_template_id, version),
            check ((kind='proof_url' and proof_url is not null and proof_text is null)
                or (kind='proof_text' and proof_text is not null and proof_url is null)))

-- evaluations: 1 per (mentor, submission), scoring + feedback + verdict
evaluations(id uuid pk, submission_id fk, evaluator_id uuid fk auth.users,
            scores jsonb, total_score numeric(6,2), feedback text,
            verdict verdict, created_at, updated_at,
            unique(submission_id, evaluator_id))

-- pitch_scores: jury scoring day 2 (5 criteria x 20 each = 100)
pitch_scores(id uuid pk, event_id fk, player_id fk, juror_id uuid fk auth.users,
             c1 smallint, c2 smallint, c3 smallint, c4 smallint, c5 smallint,
             total_score smallint generated always as (c1+c2+c3+c4+c5) stored,
             created_at, updated_at,
             check (c1 between 0 and 20 and c2 between 0 and 20 and c3 between 0 and 20
                    and c4 between 0 and 20 and c5 between 0 and 20),
             unique(event_id, player_id, juror_id))
```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| `auth.users` -> `public.*` | Authenticated user identity propagates to row-level policies via `auth.uid()` |
| `service_role` key (server) -> DB | Server actions using service role bypass RLS; only used by GameMaster import flow in Phase 4 |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | I (Information disclosure) | players, submissions, evaluations | mitigate | RLS policies gate reads on `is_my_player()` / `is_mentor()` (task 3) |
| T-01-02 | T (Tampering) | submissions.submitted_by | mitigate | Insert/update policy enforces `submitted_by = auth.uid()` (task 3) |
| T-01-03 | T (Tampering) | evaluations.evaluator_id | mitigate | Insert/update policy enforces `evaluator_id = auth.uid()` and `is_mentor()` (task 3) |
| T-01-04 | E (Elevation of privilege) | profiles.app_role | mitigate | Profile update policy: own row OR game_master only — players cannot self-promote (task 3) |
| T-01-05 | I | pitch_scores | mitigate | Players see only their own pitch scores (`is_my_player(player_id)`) (task 3) |
| T-01-06 | D (DoS) | unbounded text columns | accept | Pilot-grade; rely on Supabase per-row size limits, no rate limiting Phase 1 |
| T-01-07 | R (Repudiation) | who edited what | accept | No audit log Phase 1 (PROJECT.md Out of Scope) |
